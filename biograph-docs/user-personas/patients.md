# Patients – Daily Workflow & Navigation Guide

A Patient uses the system through the Patient Portal to manage appointments, view diagnostic reports, and track their healthcare journey.  
This page provides a simple navigation guide for common patient tasks.

## How to Use This Page
Use this page as a quick navigation guide. Each workflow represents a common patient task and shows the exact system path to follow.

---

## Core Workflow

Access healthcare services and records through the portal.

**Workflow:**  
Patient Portal → [Appointments](../clinical-workflows/booking-appointments.md) → [Diagnostics](../diagnostics/diagnostic-reports.md) → Follow-up Appointment  

## Steps Explained

- **Patient Portal** – Entry point for all patient actions  
- **Appointments** – Book and manage consultations  
- **Diagnostics** – View lab tests and reports  
- **Follow-up Appointment** – Plan next visit  

---

## Daily Jobs To Be Done

### 1. Book Appointments  
Workflow:  
Patient Portal → [Appointments](../clinical-workflows/booking-appointments.md) → Book → Select Department → Select Practitioner → Select Slot → Payment → Confirmation  

### 2. View Diagnostic Reports  
Workflow:  
Patient Portal → [Diagnostics](../diagnostics/diagnostic-reports.md) → Select Report → View Details  

### 3. Track Appointments  
Workflow:  
Patient Portal → [Appointments](../clinical-workflows/booking-appointments.md) → View Status 

### 4. Plan Follow-up Appointment  
Workflow:  
Patient Portal → [Appointments](../clinical-workflows/booking-appointments.md) → Book → Follow-up  

## Key Navigation Insight

Most patient activities revolve around a single core entry point:

- **Patient Portal** – Central access for all patient interactions  

From the portal, patients can:

- book and manage appointments  
- view diagnostic reports and results  
- track their healthcare journey  
- plan follow-up visits  

Understanding this central entry point enables patients to easily navigate and access all healthcare services.
